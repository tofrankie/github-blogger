## 为什么

`extension/panels/multi-select-input.ts` 目前把输入流程、配置写入、仓库初始化和成功/失败提示揉在一起，阅读和维护成本都很高。现在需要把这条初始化流程重新梳理清楚，并让新手也能从文档和代码中看懂它是怎么注册、怎么执行、怎么结束的。

## 变更内容

- 补充一份初始化流程说明文档，覆盖扩展注册、输入流程跳转、配置写入和仓库创建/跳过的完整路径
- 重构并重命名 `multi-select-input.ts`，把它改成更贴近职责的名称，比如 `setup-flow.ts`，同时拆开多步输入、状态处理、持久化和初始化结果提示
- 入口文件改名后，同步整理内部对应的方法、变量和类型命名，避免出现文件名与实现职责不一致的情况
- 将多步初始化改成数据驱动的步骤编排，后续如果只想调整两个步骤的先后顺序，只需要改步骤数组顺序，不需要改其他流程代码
- 保留现有配置项和交互结果，但让流程更清晰、职责更单一、后续更容易扩展
